Identify or Remove

You have two options: Only Remove and Only Identify:
Only Remove: Remove the adapters, but to do so, you must enter the adapter sequence in the fields below. In case of single read, insert only one adapter sequence, but for the paired-end file insert two adapter sequences (forward and reverse).
Only Identify: Identify the adapters without removing them. The adapters to search for are found in the 'adapters.txt' file, and you can edit it according to your needs, keeping the default structure. The result informs you the adapter found and the amount found (with up to 3 mismatches).
info

Quality Trim

Quality trimming allows you to remove regions with poor sequencing quality.
Window based quality trimming

Trim low quality regions using a sliding window based approach with the window size provided. The method is inspired from AdapterRemoval and Sickle. Two parameters (Min Quality and Quality Window Size). must be set for this.
> Min Quality: Defines the minimum average quality value of the window.
> Quality Window Size: Size of the sliding window.

The idea is to determine new start (5') and end (3') positions for the read. For this, in the direction 5' -> 3' marks as the new position 5' the first window with an average greater than minQuality the first base of the kmer must also be greater than minQuality. From the previous position, the window slides towards 3' and marks as new position 3' first window with minQuality average less than or equal. The new 3 'is placed at the bottom of that window where the quality is greater than or equal to minQuality. If no 5 'position can be determined, the reading is discarded.

Dealing with n bases

To deal with N-base removal, you have two options:
> Maximum N: Discard readings containing more than "Maximum N" bases.
> Trim flanking N bases: Removes Ns found at the edges of the read. Ex: NNNATAGATNNN -> ATAGAT
In addition, the minimum size of the reads can also be determined, after the previous checks. Using the parameter 'Minimum Read Length:
> Minimum Read Length: Discard reads smaller than the reported size.

info

Mismatch

> Max Mismatch 5'-> 3': Amount of errors allowed between the adapter and the sequence fetched in the entire sequence.
> Max Mismatch in Region 3': If the adapter with up to Max Mismatch errors is not found, it is checked if there is an adapter in the 3' end with up to "Max Mismatch in Region 3' " in order to detect interrupted adapters in the sequencing.
